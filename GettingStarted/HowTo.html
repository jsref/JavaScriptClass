Associating Javascript with an HTML page

Embedding Javascript directly in a page

Javascript can be directly included in the body of an HTML page. Code contained in that way will simply run when the page is loaded. For example, the following page will put up an alert box that says, "hello." ("alert" is built in to Javascript in the browser.) 

    
<html>
 <head>
  <title>
   Inline Javascript
  </title>
 </head>
 <body>
  <script type="text/javascript">
   alert("hello");
  </script>
 </body>
</html>

The HTML <script> element contains code to be executed by the browser.
The "type" attribute specifies that the MIME type of the script is text/javascript, so the browser knows to interpret the contents of the element as javascript. Javascript is standard across browsers. Some browsers understand other scripting languages. Try it here

Delaying execution

Generally you will not want to execute a script immediately. To defer the execution to a later time, and to make your programs more modular, you can group your actions into "functions." There are several ways to do this, but the simplest is to use the function keyword: 

    
<script type="text/javascript">
    function say_hello() {
        alert("hello");
    }
</script>

Because functions are used to define code to be run at a later time it is conventional to define them in the head of an HTML page rather than in the body. The page in the following example exhibits the same behavior as the first example but it does so by:

  1. defining the function named "say_hello" in the head of the page
  2. invoking the function from the body of the page
    
<html>
 <head>
  <title>
   Inline Javascript
  </title>
  <script type="text/javascript">
   function say_hello() {
      alert("hello");
   }
  </script>
 </head>
 <body>
  <script type="text/javascript">
   say_hello();
  </script>
 </body>
</html>

Separating Javascript from HTML

While it is preferable to define code in functions for later use, it would be better yet to put that code somewhere from which it could be used by more than one web page. To do this we use the "src" attribute of the script tag. The value of src is the name of the file containing our Javascript code. For example, we could put the following code in a file named hello.js: 

    
function say_hello() {
    alert("hello");
}

We can use it in our page by including the following code in the head of the page: 

    
<html>
 <head>
  <title>
   Inline Javascript
  </title>
  <script type="text/javascript" src="hello.js"></script>
  </script>
 </head>
 <body>
  <script type="text/javascript">
   say_hello();
  </script>
 </body>
</html>

Try it here

Note that when Javascript code appears in a file by itself, rather than being embedded in an HTML page you do not enclose it in a <script> element.

Testing Javascript in a file without a web server

Many web browsers will allow you to load an HTML file locally by selecting something like:

Alternatively, local files can be specified by urls. For example, if your home directory where /Users/puppy and you were working on a file named hello.html in the html subdirectory, you could access that file by typing the following into the location bar of your browser:
file:///Users/puppy/html/hello.html

Because Javascript is interpreted as part of a web page by the browser, you can simply load files containing Javascript to try things out and make sure your code is working. You only need to use a web server if something in your web page needs to communicate with a service by sending data via AJAX or submitting a form.

Event Handling

Javascript in a web page is typically run in response to an action on the part of the person viewing the page. In the last example for this session we will associate the say_hello function with a link in the page.

Some HTML elements have attributes that correspond to actions taken using a mouse. The anchor element, <a>, supports an "onclick" attribute. So if we wanted to put up an alert that says, "hello," every time a link was clicked we could:

  1. add the function say_hello to the head of the page
  2. add an anchor element to the page
  3. add a call to say_hello to the onclick attribute of the anchor
    
     <a href="#" onclick="say_hello()">Hello</a>
